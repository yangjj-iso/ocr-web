# 人社局需求整改优先级清单

更新时间：2026-04-14  
优先级规则：按“是否阻塞严格验收”排序，不按实现难度排序。

## P0：当前验收阻塞项

### P0-1 结构化页 OCR 链路回正
- 对应需求：`10`，同时影响 `02`、`05`、`14`、`15`
- 当前问题：
  - 分件核对页样本只返回 `lines`
  - `regions/table_data/seal` 没有稳定进入页面
  - 表格、印章、正文无法按结构化方式区分
  - 预览页仍是倾斜扫描图
- 实施方向：
  - Worker 页 OCR 改为“结构化优先、纯文本兜底”
  - Review API 保证 `ocr_pages` 返回 `regions/table_data/region_lines`
  - 分件页默认展示结构化区域，不再把全页文本行当主视图
  - 将印章区域降为单独证据项，不混入正文主区域
  - 增强页预处理和矫正策略，优先给审核页返回更适合人工核对的预览图
- 验收标准：
  - 当前问题样本页能返回非空 `regions`
  - 至少有 1 个 `table` 区域
  - 印章以 `seal` 或独立证据块展示
  - 分件页默认不再刷满全页蓝框
  - 样本页在人工核对界面中方向正确或明显优于当前版本

### P0-2 归档章识别到自动分件闭环
- 对应需求：`10`
- 当前问题：
  - 归档章识别能力零散存在，但没有形成稳定的自动分件产品链路
  - 无法证明“混合批次多件档案自动分件”可用
- 实施方向：
  - 定义归档章抽取字段：全宗号、年度、保管期限、件号
  - 设计分件决策优先级：归档章 > 结构化规则 > 文件名线索 > 人工队列
  - 对无法识别或冲突样本统一转人工处理队列
- 验收标准：
  - 给定混合批次样本，系统能自动切成多件
  - 每件都能关联件号或完整档号
  - 失败页进入人工队列并带原因

### P0-3 5 级目录生成与图片挂接
- 对应需求：`08`、`09`、`11`
- 当前问题：
  - `storage_path` 字段存在但样本为空
  - 未形成物理目录生成与图片归位能力
- 实施方向：
  - 落地目录规则引擎：根目录 / 全宗 / 年度 / 保管期限 / 件号
  - 生成实体目录并回写 `storage_path`
  - 上传或自动分件完成后，把图片复制或移动到目标目录
- 验收标准：
  - 批量样本能落到实际 5 级目录
  - 档案记录 `storage_path` 非空且可打开
  - 图片和目录一一对应

### P0-4 双层 PDF 导出
- 对应需求：`21`
- 当前问题：
  - 前端有调用占位，后端接口不可用
  - 没有“原图层 + 隐藏文字层”的双层 PDF
- 实施方向：
  - 以页图为底层，按 OCR/结构化结果生成隐藏文字层
  - 支持单件、多件、按检索结果导出
  - 文件名按档号规则命名
- 验收标准：
  - 导出 PDF 可检索、可复制文本
  - 页面顺序正确
  - 文件名符合规则
  - 接口和前端按钮都可用

### P0-5 Excel 模板导入
- 对应需求：`01`
- 当前问题：
  - 只有导出，没有导入
- 实施方向：
  - 定义模板字段映射与校验
  - 支持导入后回写到批次或档案记录
  - 对错误行给出清晰反馈
- 验收标准：
  - 可导入人社局模板样式
  - 导入结果可继续进入审核、检索和导出链路

## P1：强烈建议在验收前补齐

### P1-1 批量字段修改
- 对应需求：`04`
- 方向：
  - 后端提供按条件批量更新接口
  - 前端提供按年度、保管期限、密级等批量替换界面
- 验收标准：
  - 能批量替换责任者、密级、备注等字段

### P1-2 低置信度高亮与修改日志
- 对应需求：`05`
- 方向：
  - 将 OCR/结构化结果中的置信度透出到前端
  - 低置信度字符或字段标记为需人工关注
  - 逐字段修改写入可追溯日志
- 验收标准：
  - 审核员能快速定位低质量字段
  - 修改前后快照可查询

### P1-3 档案层检索补强
- 对应需求：`13`、`14`、`15`、`17`
- 当前问题：
  - 任务层能搜，档案层命中不稳定
  - 结果没有高亮
  - 高级筛选不完整
- 方向：
  - 先修档案层关键字命中与字段映射
  - 再补高亮、文号/密级/年度/保管期限筛选
  - 最后再考虑倒排索引
- 验收标准：
  - 已归档记录可稳定命中关键字
  - 结果展示关键词高亮和路径信息
  - 支持按需求表字段做二次过滤

### P1-4 操作日志覆盖面
- 对应需求：`19`
- 当前问题：
  - 日志接口返回空数据
  - 未覆盖查看、检索、著录修改等动作
- 方向：
  - 统一前后端关键动作埋点
  - 补齐 `query/view/edit/delete/review/export` 行为记录
  - 确保带操作人、时间、IP、对象和摘要
- 验收标准：
  - 运行一轮著录和检索后，日志不为空
  - 能按动作类型和用户查询

## P2：中期增强项

### P2-1 词库管理
- 对应需求：`20`
- 方向：
  - 提供自定义词库管理 UI
  - 将词库接入 OCR 和字段提取链路
- 验收标准：
  - 可增删改专业术语、人名、机构简称

### P2-2 规则配置产品化
- 对应需求：`07`
- 方向：
  - 从“策略 JSON 快照”升级到可视化规则管理
  - 让档号规则、目录规则、分隔符、路径模板都能配置
- 验收标准：
  - 管理员无需改代码即可调整规则

### P2-3 预览与定位体验增强
- 对应需求：`16`
- 方向：
  - 在线预览继续加强
  - 增加直接定位实体文件夹能力
- 验收标准：
  - 从检索结果可直接跳到路径或打开目录

## 推荐整改顺序

1. 先修 `P0-1 结构化页 OCR 链路回正`
2. 再做 `P0-2 自动分件闭环`
3. 同步落地 `P0-3 目录生成与路径回写`
4. 并行补 `P0-4 双层 PDF 导出`
5. 最后补 `P0-5 Excel 模板导入`

## 管理层结论

- 如果目标是“尽快做成一个可给人社局演示的版本”，优先级必须是：
  - 自动分件可看
  - 目录可落
  - 双层 PDF 能导
  - 关键日志可查
- 如果这四块不补，系统即使 OCR 能跑、页面能看，也不适合进入正式验收。
